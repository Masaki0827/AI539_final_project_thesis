# AI Interview Coach: Optimizing Software Engineering Interview Feedback through RLAIF and DPO

This repository contains the LaTeX source and associated files for the AI539 final report on an AI-driven interview simulation and feedback system.

## Project Overview
Preparing for software engineering interviews is a critical but resource-intensive process. This project presents an **AI Interview Coach** based on the Qwen2.5-3B-Instruct model, optimized through **Reinforcement Learning from AI Feedback (RLAIF)** and **Direct Preference Optimization (DPO)**.

### Key Features
- **Specialized Coaching:** Tailored feedback for technical communication and algorithmic explanations.
- **RLAIF & DPO:** Fine-tuned using a high-quality preference dataset generated by a stronger judge model (Qwen3.5-9B).
- **Simulated Personas:** Trained on a diverse range of student failure modes (vague, verbose, or partially correct answers).

## Methodology
The optimization process involved four main phases:
1. **Dataset Construction:** 4,853 software engineering interview questions.
2. **Student Simulation:** Generating diverse response modes using base LLMs.
3. **Preference Labeling (RLAIF):** Using Qwen3.5-9B to judge and rank feedback quality.
4. **Model Optimization:** Utilizing QLoRA and DPO for efficient fine-tuning.

## Status
The project implementation is currently in the **Model Optimization** phase. The report methodology and baseline results are documented in `main.tex`. Final experimental results will be added upon completion of the DPO training phase.
